400 /**
401  * Enables or disables Multipart file extraction. This function can be invoked only
402  * after a previous htp_config_set_tmpdir() invocation. Otherwise, the configuration
403  * change will fail, and extraction will not be enabled. Disabled by default. Please
404  * note that the built-in file extraction implementation uses synchronous I/O, which
405  * means that it is not suitable for use in an event-driven container. There's an
406  * upper limit to how many files can be created on the filesystem during a single
407  * request. The limit exists in order to mitigate against a DoS attack with a
408  * Multipart payload that contains hundreds and thousands of files (it's cheap for the
409  * attacker to do this, but costly for the server to support it). The default limit
410  * may be pretty conservative.
411  *
412  * @param[in] cfg
413  * @param[in] extract_files 1 if you wish extraction to be enabled, 0 otherwise
414  * @param[in] limit the maximum number of files allowed; use -1 to use the parser default.
415  */
