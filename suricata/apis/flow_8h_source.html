282 /**
283  * \brief Flow data structure.
284  *
285  * The flow is a global data structure that is created for new packets of a
286  * flow and then looked up for the following packets of a flow.
287  *
288  * Locking
289  *
290  * The flow is updated/used by multiple packets at the same time. This is why
291  * there is a flow-mutex. It's a mutex and not a spinlock because some
292  * operations on the flow can be quite expensive, thus spinning would be
293  * too expensive.
294  *
295  * The flow "header" (addresses, ports, proto, recursion level) are static
296  * after the initialization and remain read-only throughout the entire live
297  * of a flow. This is why we can access those without protection of the lock.
298  */
299
