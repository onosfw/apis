31 /** When the ringbuffer is full we have two options, either we spin & sleep
32  * or we use a pthread condition to wait.
33  *
34  * \warning this approach isn't working due to a race condition between the
35  * time it takes for a thread to enter the condwait and the
36  * signalling. I've obverved the following case: T1 sees that the
37  * ringbuffer is empty, so it decides to start the wait condition.
38  * While it is acquiring the lock and entering the wait, T0 puts a
39  * number of items in the buffer. For each of these it signals T1.
40  * However, as that thread isn't in the "wait" mode yet, the signals
41  * are lost. T0 now is done as well and enters it's own wait
42  * condition. T1 completes it's "wait" initialization. It waits for
43  * signals, but T0 won't be able to send them as it's waiting itself.
44  */

47 /** \brief ring buffer api
48  *
49  * Ring buffer api for a single writer and a single reader. It uses a
50  * read and write pointer. Only the read ptr needs atomic updating.
51  */

55  SC_ATOMIC_DECLARE(unsigned char, write); /**< idx where we put data */
56  SC_ATOMIC_DECLARE(unsigned char, read); /**< idx where we read data */
