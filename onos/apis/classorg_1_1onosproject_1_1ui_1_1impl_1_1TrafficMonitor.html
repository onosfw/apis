synchronized void org.onosproject.ui.impl.TrafficMonitor.monitor ( Mode  mode)

Monitor for traffic data to be sent back to the web client, under the given mode.

This causes a background traffic task to be scheduled to repeatedly compute and transmit the appropriate traffic data to the client.

The monitoring mode is expected to be one of:

  • ALL_FLOW_TRAFFIC
  • ALL_PORT_TRAFFIC
  • SELECTED_INTENT

modemonitoring mode

synchronized void org.onosproject.ui.impl.TrafficMonitor.monitor ( Mode  mode,
NodeSelection  nodeSelection 
)

Monitor for traffic data to be sent back to the web client, under the given mode, using the given selection of devices and hosts.

In the case of "device link flows", this causes a background traffic task to be scheduled to repeatedly compute and transmit the appropriate traffic data to the client. In the case of "related intents", no repeating task is scheduled.

The monitoring mode is expected to be one of:

  • DEV_LINK_FLOWS
  • RELATED_INTENTS
