void org.onosproject.net.proxyarp.impl.ProxyArpManagerTest.testReplyDifferentVlan ( )

Tests ProxyArpManager#reply(Ethernet, ConnectPoint) in the case where the destination host is known for that IP address, but is not on the same VLAN as the source host.

An host is connected on device 6, port 1 where no interfaces are defined. It sends ARP requests from VLAN10, not configured anywhere in the network. Another host with the IP address requested lives on device 5, port 1 in the network. Anyway, since the host uses another VLAN it's not found and the ARP packet is flooded out of port 4 and 5.

Verifies the ARP request is flooded out the correct edge ports.

void org.onosproject.net.proxyarp.impl.ProxyArpManagerTest.testReplyKnown ( )

Tests ProxyArpManager#reply(Ethernet, ConnectPoint) in the case where the destination host is known.

Two host using the same VLAN are registered on the host service on devices 5 and 6. Host on port 6 asks for the MAC of the device on port 5. Since the destination mac address is known, the request is not flooded to anywhere and ONOS directly builds an ARP reply, sended back to the requester on device 6. It's verified that a proper ARP reply is received on port 1 of device 6.
